About this site

How's My SSL? is a cute little website that's meant to tell you how your TLS client (like the web browser you're reading this with) is doing.

How's My SSL? was borne out of my own need to know more about TLS clients in the wild while I worked on a web server at my day job. It's purpose is to give developers and technically-savvy folks a quick an easy way to interrogate their tools and learn more about them.

It's also meant to put impetus on developers to improve their tools by modernizing their TLS stacks. While it can't possibly detect every problem is a TLS client, it can check for many common ones.

Rating

How's My SSL? gives a rating to each client that connects to it. These ratings are not the final word on "how secure" a client is, but merely captures some important aspects of the client.

Clients are rated as "Probably Okay" if no security problems could be detected.

Clients are downgraded to "Improvable" if they do not support ephemeral key cipher suites, or session tickets.

Clients are downgraded to "Bad" if any of the following are true:

  • It uses TLS 1.0 (instead of 1.1 or 1.2), or, worse, SSLv3.
  • It supports known insecure cipher suites. Developers can find the list of known insecure cipher suites in the howsmyssl repository on GitHub.
  • It supports TLS compression (that is compression of the encryption information used to secure your connection) which exposes it to the CRIME attack.
  • It is susceptible to the BEAST attack. A caveat here: the first is that some advanced clients have a workaround called "1/n-1 record splitting" that is not yet detectable by Hows My SSL?. Known clients that do this are recent versions of Chrome and Firefox but it is unlikely that others do.

(By the way, clients rated as "Bad" may also have problems that, on their own, would have the client rated as "Improvable".)

TLS vs SSL

So, the jargon around these things is weird. When folks refer to the "TLS" they are referring to the modern protocols of encrypting data across the internet. "SSL", when used by experts, refers to the first versions of these modern protocols.

For Developers

How's My SSL? provides a simple HTTP JSON API for developers that want to test clients without a browser experience: https://howsmyssl.com/a/check.

    GET https://howsmyssl.com/a/check

    {
      given_cipher_suites: [
        "TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256",
        "TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256",
        "TLS_DHE_RSA_WITH_AES_128_GCM_SHA256",
        "TLS_RSA_WITH_AES_128_GCM_SHA256",
        "TLS_ECDHE_ECDSA_WITH_AES_256_CBC_SHA",
        "TLS_ECDHE_RSA_WITH_AES_256_CBC_SHA",
        "TLS_DHE_RSA_WITH_AES_256_CBC_SHA",
        "TLS_RSA_WITH_AES_256_CBC_SHA",
        "TLS_ECDHE_ECDSA_WITH_RC4_128_SHA",
        "TLS_ECDHE_ECDSA_WITH_AES_128_CBC_SHA",
        "TLS_ECDHE_RSA_WITH_RC4_128_SHA",
        "TLS_ECDHE_RSA_WITH_AES_128_CBC_SHA",
        "TLS_DHE_RSA_WITH_AES_128_CBC_SHA",
        "TLS_DHE_DSS_WITH_AES_128_CBC_SHA",
        "TLS_RSA_WITH_RC4_128_SHA",
        "TLS_RSA_WITH_RC4_128_MD5",
        "TLS_RSA_WITH_AES_128_CBC_SHA",
        "TLS_RSA_WITH_3DES_EDE_CBC_SHA",
        "TLS_DHE_DSS_EXPORT_WITH_DES40_CBC_SHA"
      ],
      ephemeral_keys_supported: true,
      session_ticket_supported: true,
      tls_compression_supported: false,
      unknown_cipher_suite_supported: false,
      beast_attack_vuln: false,
      insecure_cipher_suites: {
        "TLS_DHE_DSS_EXPORT_WITH_DES40_CBC_SHA": [
          "uses keys smaller than 128 bits keys in its encryption"
        ]
      },
      tls_version: "TLS 1.2",
      rating: "Bad"
    }

The returned JSON hash will have a least these keys, but may include more as the the security environment changes. Each of these values keys map fairly obviously to the HTML version and the about its text can be helpful to understand their meaning. The source code that gathers the client info may also be helpful. All strings returned may contain HTML escaped text.
